# 호환성 프로필 (고정 기준선)

이 문서는 편집기가 `random-exam-generator` 호환성 경고/오류를 판단할 때
사용하는 고정 기준선을 정의합니다.

## 1) 컴파일 환경

- 컴파일 명령: `latexmk -pdf -interaction=nonstopmode`
- 엔진 기준: `pdflatex`
- 기준 출처:
  - 생성기 호환성 정책(외부 버전 관리)
  - 편집기 내부 고정 프로필(진단 일관성 보장)

## 2) 수식 패키지 플래그

사용자에게 노출하는 진단에는 수식 관련 패키지 상태만 포함합니다.
수식과 무관한 패키지 목록은 의도적으로 제외합니다.

- `amsmath`: `false`
- `amssymb`: `false`
- `amsfonts`: `false`
- `mathtools`: `false`

이 값은 생성기 preamble 정책이 바뀔 때만 변경하는 것을 원칙으로 합니다.

## 3) 진단 UX 정책

- 사용자 오류/경고는 입력창 원문(사용자 입력) 기준으로 표시합니다.
- 내부 변환 문자열은 검사 로직에서만 사용합니다.
- 진단 메시지는 가능한 한 다음을 우선합니다:
  - 입력 원문 명령/토큰(예: `\co`, `\underline{...}`)
  - 필요한 경우 짧은 원문 스니펫(`입력식: ...`)
- 이스케이프 표기는 사용자 입력 형태를 유지합니다(예: `\co`, `\\co` 아님).

## 4) 범위 분리

- 지문(`text`): `pdflatex` 컴파일 호환성 검사
- 정답식(`answer`): 생성기 계산 파서 호환성 검사

UI에서는 함께 보여주더라도, 필드 성격에 맞는 규칙으로 각각 판정합니다.

## 5) 툴바 정책 기준

- 정답식 모드 버튼 허용/차단은 반드시
  `RandomExamCompat.evaluateAnswerToolbarAction(...)`를 사용합니다.
- UI는 버튼 원본 메타데이터(`action`, `cmd`, `tpl`, `prefix`, `suffix`, `policyKey`)를 그대로 전달합니다.
- `app.js`에 파서 호환 명령 집합을 중복 정의하지 않습니다.

## 6) 명령 분류와 패키지 의존성

- LaTeX 명령은 두 축으로 분리 관리합니다.
  - 모드 분류: `수식 전용` / `텍스트 전용` / `공통`
  - 패키지 의존: 현재 preamble에서 실제 사용 가능한지 여부
- 예: `\mathbb`는 모드상 수식 명령이지만, 현재 기준(AMS 미포함)에서는 패키지 의존 경고를 냅니다.
- 이 분리는 “모드 경고”와 “컴파일 환경 경고”를 유지보수 관점에서 독립적으로 업데이트하기 위한 기준입니다.
- 기본 원칙은 컴파일 기준입니다.
  - 수식 내부에서도 실제 pdflatex 컴파일이 가능한 명령(예: `\textbf`, `\textit`, `\texttt`, `\emph`, `\newline`)은 오류성 경고에서 제외합니다.
  - 실제 실패 가능성이 높은 항목(예: `\mathbb` 패키지 의존, `\par`의 수식 내부 사용)은 경고를 유지합니다.
