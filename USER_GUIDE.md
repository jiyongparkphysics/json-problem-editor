# 문제 JSON 편집기 사용자 가이드

## 1) 개요
- 이 도구는 시험 문항 JSON을 작성/수정하고, 이미지 포함 ZIP으로 저장/불러오기 할 수 있는 브라우저 기반 편집기입니다.
- 실행은 `index.html`을 브라우저에서 여는 방식입니다.
- 최소 권장 화면 폭은 1024px 이상입니다.

## 2) 빠른 시작
1. `index.html`을 브라우저에서 엽니다.
2. 문항의 `ID`, `제목`, `지문`, `정답식`, `배점`을 입력합니다.
3. 지문에서 `*var*` 형식으로 변수를 넣으면 변수 정보가 자동 생성됩니다.
4. 필요하면 이미지를 첨부합니다.
5. `저장하기(JSON/ZIP)`로 내보냅니다.

## 3) 기본 작성 규칙
- `ID`는 문항마다 고유해야 합니다.
- `ID`는 저장 시 앞뒤 공백이 제거됩니다.
- 변수는 지문의 `*var*` 표기를 기준으로 관리됩니다.
- 지문 미리보기에서 변수(`*var*`)를 파란 굵은 글씨로 보려면, `*var*`를 `$...$` 또는 `$$...$$` 안에 넣는 것을 권장합니다. 예: `$*a*$`, `$$*a*+*b*$$`
- `정답식`에 쓰는 변수 이름은 변수 정보 목록과 일치해야 합니다.
- 이미지는 문항당 1개를 첨부할 수 있으며, 5MB 초과 이미지는 불러오기에서 제외됩니다.

## 4) 저장/불러오기
- JSON 저장: 문항 데이터만 저장합니다.
- ZIP 저장: 문항 JSON + 이미지 데이터를 함께 저장합니다.
- ZIP 불러오기:
  - 먼저 문항 목록을 보여주고,
  - `전체 교체` 또는 `선택 추가` 실행 시 실제 반영됩니다.

## 5) 자동저장/복구
- 문항 메타데이터는 `localStorage`에 자동저장됩니다.
- 이미지 스냅샷은 `IndexedDB`에 저장됩니다.
- `새로시작`을 누르면 자동저장 데이터와 이미지 스냅샷이 함께 초기화됩니다.

## 6) random-exam-generator 호환 작성 가이드
`random-exam-generator`와 호환하려면 아래를 지키는 것을 권장합니다.

### 6-1. 변수 이름 규칙 (중요)
- 지문 변수 표기는 `*var*` 형식을 사용합니다.
- 미리보기 강조를 위해 변수 표기는 수식 구분자 안에서 사용하는 것을 권장합니다. 예: `$*v0*$`, `$$*v0*+*v1*$$`
- 호환 안전 규칙: 변수명은 `영문 시작 + 영문/숫자`만 사용하세요. 예: `v0`, `mass1`
- `_`(언더스코어)는 피하세요. 편집기에서는 허용되지만 생성기의 지문 치환 정규식과 불일치가 날 수 있습니다.

### 6-2. 정답식(LaTeX) 규칙
- 정답식에서는 변수에 `*`를 붙이지 말고 변수명만 사용하세요. 예: `\frac{v0^{2}}{2*g}`
- 정답식 칸은 이미 "수식 문맥"이므로 `$...$` 구분자를 넣지 않는 것을 권장합니다.
- `\left`, `\right`는 괄호 `()` 또는 `[]` 문맥에서만 사용하세요.
- 곱셈은 반드시 `*`로 명시하세요. 예: `2*x`, `a*b`, `x*(y+1)`
- 아래 명령 위주로 작성하세요:
  - `\frac`, `\sqrt`
  - `\sin`, `\cos`, `\tan`, `\sinh`, `\cosh`, `\tanh`
  - `\arcsin`, `\arccos`, `\arctan`, `\tan^{-1}`
  - `\ln`, `\exp`, `\pi`
  - 거듭제곱은 `^{...}` 형태 권장 (예: `x^{4}`)
  - 로그는 `\ln(x)`만 사용하세요. `\log(x)`, `\log_{a}(x)`는 호환성 오류 대상입니다.

### 6-2-1. 지문의 그리스 문자(유니코드) 주의
- 지문에 `α`, `β` 같은 그리스 유니코드를 직접 입력하면 KaTeX 미리보기에서는 보여도, `pdflatex` 기반 출력에서 글리프 누락/경고가 날 수 있습니다.
- 지문에서는 수학 명령을 반드시 수식 구분자로 감싸세요. 예: `\alpha`(X), `$\alpha$`(O)
- 편집기는 이 경우 호환성 `경고`를 표시하며, 경고 박스의 `자동치환` 버튼으로 문맥에 맞게 치환합니다.
  - 수식 밖 텍스트: `α -> $\alpha$`
  - 이미 `$...$` 또는 `$$...$$` 안: `α -> \alpha`
- 자동치환 시 명령 뒤에 영문자가 바로 붙는 경우에는 구문 충돌 방지를 위해 필요 시 `{}`가 함께 들어갑니다. 예: `αb` -> `\alpha{}b`

### 6-2-2. 지문/정답식 수식 모드 차이
- 지문: 일반 텍스트 문맥이 기본이므로 수학 표현은 `$...$`(인라인 수식) 또는 `$$...$$`(블록 수식)로 감싸야 합니다.
- 정답식: 편집기/생성기에서 수식으로 처리되므로 `$...$` 없이 LaTeX 수식만 입력하세요.
- 편집기는 지문에서 `\alpha`, `\frac`, `\sin` 같은 수식 전용 명령이 `$...$` 밖에 있으면 호환성 경고를 표시합니다.
- 경고 박스의 `자동치환` 버튼으로 해당 명령을 `$...$`로 감싸는 자동 보완을 적용할 수 있습니다.
- `\underline{...}` 같은 텍스트/서식 명령은 지문에서는 사용할 수 있지만, 정답식(계산식)에서는 호환성 오류로 처리됩니다.

### 6-2-3. 지문 특수문자 이스케이프
- `%`, `&`, `#`는 지문에서 이스케이프 없이 쓰면 `pdflatex` 컴파일 실패 가능성이 높으므로 `\%`, `\&`, `\#`로 입력하세요.
- `_`, `^`, `~`는 지문 텍스트(수식 구분자 밖)에서 그대로 쓰면 오류/깨짐 가능성이 있어 `\_`, `\^{}`, `\~{}`를 권장합니다.
- 편집기는 위 문자를 감지하면 호환성 경고와 `자동 이스케이프` 자동 수정 버튼을 제공합니다.
- 단, `$...$`/`$$...$$` 수식 안의 `_`, `^`는 일반 수식 문법으로 사용될 수 있어 자동 이스케이프 대상에서 제외됩니다.

### 6-2-4. 유니코드 위/아래첨자 주의
- 지문에 `²`, `₁` 같은 유니코드 위/아래첨자를 직접 입력하면 환경에 따라 출력이 불안정할 수 있습니다.
- 편집기는 해당 문자를 경고로 안내하고 `자동치환` 버튼을 제공합니다.
- 자동치환 시:
  - 수식 밖: `x² -> $x^{2}$`, `y₁ -> $y_{1}$`
  - 수식 안: `$x²$ -> $x^{2}$`

### 6-3. 피해야 할 표현
- 생성기에서 실패 가능성이 큰 표현:
  - `\log_{a}(x)` (밑 있는 로그)
  - `\log(x)` (현재 파서 미지원)
  - `\sec`, `\csc`, `\cot`, `\asin`
  - `\mathrm{e}` 같은 서식/폰트 명령을 계산식에 섞는 방식
  - `x^4` 같은 일반 캐럿 지수 표기(`^{...}` 대신)와 `e^{x}` 표기
  - `2x`, `ab`, `x(y+1)` 같은 묵시적 곱셈
  - 과도한 LaTeX 장식 명령(계산과 무관한 명령)
- 아래처럼 단순식으로 바꾸는 것이 안전합니다:
  - `\ln(x)`
  - 지수는 `x^{n}`, 지수함수는 `\exp(x)`
  - 변수명은 `x1`처럼 평문 식별자, 곱셈은 `*`로 명시

### 6-4. 이미지 파일 규칙
- 생성기는 ZIP 내부 이미지 확장자를 `.png`로 처리합니다.
- 이미지 파일명은 반드시 문항 `id`와 정확히 일치해야 합니다.
  - 예: 문항 `id`가 `Q1`이면 이미지 파일은 `Q1.png`

### 6-5. 호환 기준 LaTeX/PDF 환경
- 본 가이드는 `random-exam-generator` 기준으로 작성되었습니다.
- PDF 컴파일: `latexmk -pdf -interaction=nonstopmode`
- 실행 환경 의존성 예시(`packages.txt`): `latexmk`, `texlive-latex-extra`, `texlive-lang-korean`, `texlive-fonts-recommended`
- 운영 환경에 따라 추가 패키지가 설치될 수 있지만, 이 편집기의 호환 기준 엔진은 `pdflatex`입니다.
- 즉, KaTeX 미리보기와 실제 PDF 컴파일 결과가 다를 수 있으므로, 최종 기준은 위 LaTeX 컴파일 환경입니다.
- 편집기 미리보기는 혼동을 줄이기 위해 지문 텍스트에서 일부 LaTeX 텍스트 명령(`\\`, `\newline`, `\textbf{...}` 등)을 HTML로 근사 표시하지만, 모든 LaTeX 패키지 명령을 완전 재현하지는 않습니다.
- 줄바꿈 규칙은 LaTeX에 맞춰서 미리보기됩니다: 일반 엔터 1번은 줄바꿈이 아니라 공백으로 처리되고, 빈 줄(연속 개행)만 문단 구분으로 표시됩니다. 강제 줄바꿈은 `\\` 또는 `\newline`을 사용하세요.

## 7) 제출 전 체크리스트
1. 중복 `ID`가 없는지 확인
2. 모든 문항에 `ID/지문/정답식/변수 범위/배점`이 채워졌는지 확인
3. 정답식 변수와 변수 목록이 일치하는지 확인
4. (호환 필요 시) 변수명을 `영문 시작 + 영문/숫자` 규칙으로 통일
5. (호환 필요 시) 이미지 파일명이 `id.png` 규칙을 따르는지 확인

## 호환성 기준 프로필 (추가)

- 편집기 경고/오류는 `COMPATIBILITY_PROFILE.md`에 정의된 고정 기준을 따릅니다.
- 컴파일 기준: `latexmk -pdf -interaction=nonstopmode` (`pdflatex`).
- 사용자 경험(UX) 원칙: 경고/오류 메시지는 내부 변환식이 아니라 입력창 원문(사용자 입력) 기준으로 표시합니다.

## 호환성 메모 (AMS)

- 호환성 안내 메시지에는 AMS 수식 패키지 상태만 노출합니다.
- 현재 기준선에서는 `amsmath`, `amssymb`, `amsfonts`, `mathtools`를 미포함으로 간주합니다.
- 수식과 무관한 패키지 목록은 사용자 안내에서 의도적으로 제외합니다.
- 진단 메시지는 내부 변환식이 아니라 입력창 원문(사용자 입력) 기준으로 보여야 합니다.

## 툴바 메모 (최신)

- 정답식 모드에서 툴바 활성/비활성은 파서 호환성 규칙으로 제어됩니다.
- 장식 명령 `\overline{...}`를 지원합니다. (텍스트 용도는 `$...$` 수식 구분자 안에서 사용 권장)
- `\underline{...}` 빠른 삽입은 글꼴/기호 탭의 밑줄 그룹에서 제공합니다.
- 연산 탭에서 `\sum`, `\prod`는 `합/곱 기호` 그룹으로 분리되어 있습니다.
- 툴바 탭 키보드 이동을 지원합니다: 왼쪽/오른쪽 화살표 키, `Home/End`, `Enter/Space`.
