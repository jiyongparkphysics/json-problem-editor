# Manual Test Checklist

## 1) 이미지 자동복구 (핵심)
1. 문항 1개를 만들고 ID를 `Q1`로 입력한다.
2. 이미지(5MB 이하)를 첨부한다.
3. 페이지를 새로고침한다.
4. 같은 문항에 이미지가 그대로 보이는지 확인한다.

Expected:
- 이미지가 유지되어야 한다.

## 2) 이미지 변경 후 복구
1. 같은 문항에서 기존 이미지를 다른 이미지로 교체한다.
2. 2초 이상 대기 후 새로고침한다.
3. 교체한 이미지가 복구되는지 확인한다.

Expected:
- 이전 이미지가 아니라 "새 이미지"가 복구되어야 한다.

## 3) 복수 문항 이미지 복구
1. 문항 2~3개를 만들고 각각 다른 ID/이미지를 넣는다.
2. 새로고침한다.
3. 각 문항의 이미지가 정확히 대응되는지 확인한다.

Expected:
- 문항별 이미지 매칭이 바뀌지 않아야 한다.

## 4) ID 정규화(공백) 저장
1. ID를 `  Q1  `처럼 앞뒤 공백 포함으로 입력한다.
2. 저장하기(JSON/ZIP)를 실행한다.
3. 저장된 JSON의 `id` 값을 확인한다.

Expected:
- 저장 값은 `Q1`(trim)이어야 한다.

## 5) ID 중복 검증
1. 두 문항의 ID를 둘 다 `Q1`로 맞춘다.
2. 저장하기를 누른다.

Expected:
- 저장이 차단되고 중복 ID 검증 메시지가 보여야 한다.

## 6) 대소문자 정책 확인
1. 문항 ID를 `Q1`과 `q1`로 각각 입력한다.
2. 저장하기를 누른다.

Expected:
- 서로 다른 ID로 처리되어 저장 가능해야 한다.

## 7) 커스텀 모달 통일
1. 문항 삭제, 이미지 5MB 초과, 저장 오류 상황을 각각 유도한다.

Expected:
- 브라우저 기본 alert/confirm이 아니라 프로젝트 커스텀 모달이 표시되어야 한다.

## 8) 최소 너비 안내 배너
1. 전체 미리보기/불러오기 모달을 연 상태에서 브라우저 폭을 1024px 미만으로 줄인다.

Expected:
- 모달보다 "넓은 화면이 필요합니다" 안내가 우선 보이고, 작업 UI는 숨겨져야 한다.

## 9) 새로시작 완전 초기화
1. 문항/이미지를 하나 이상 입력한다.
2. `새로시작`을 실행한다.
3. 페이지를 새로고침한다.

Expected:
- 이전 문항/이미지가 복구되지 않아야 한다.
- 초기 빈 문항 상태로 시작되어야 한다.

## 10) 이미지 유지 + 메타 변경 후 복구
1. 이미지가 포함된 문항 1개를 만든다.
2. 이미지는 그대로 두고 지문/정답/변수/배점 중 하나만 수정한다.
3. 2초 이상 대기 후 새로고침한다.

Expected:
- 수정한 메타데이터가 유지되어야 한다.
- 이미지도 함께 복구되어야 한다.

## 11) 가져오기 후 정답 범위 패널 초기화
1. 전체 문항 점검을 실행해 결과 패널을 띄운다.
2. 불러오기에서 `전체 교체` 또는 `선택 추가`를 실행한다.

Expected:
- 기존 전체 점검 결과 패널(`minmax-all-result`, `minmax-report-table`)이 숨겨져야 한다.
- 새 데이터 기준으로 다시 점검하기 전에는 이전 결과가 남아 있지 않아야 한다.

## 12) 인쇄 시 하단 라이선스 링크 숨김
1. 전체 미리보기 모달을 연다.
2. 인쇄 버튼을 눌러 인쇄 미리보기를 확인한다.

Expected:
- 문항 본문만 인쇄 대상에 포함되고, 하단 `서드파티 라이선스 고지` 링크는 출력되지 않아야 한다.

## 13) 미리보기 드롭다운 라벨/폭 통일
1. 전체 미리보기를 연다.
2. 상단 문항 드롭다운 버튼의 폭과 라벨 표시를 확인한다.

Expected:
- 버튼 폭이 고정(`7rem`)되어 메인 드롭다운과 동일한 폭감을 가져야 한다.
- 라벨이 `번호` 형식(`n번`)으로 표시되어야 한다.

## 14) 저장 버튼 전역 검증 상태 반영
1. 문항 2개를 만든다.
2. 1번 문항은 정상 상태, 2번 문항은 오류 상태(예: ID 비움)로 둔다.
3. 다시 1번 문항으로 돌아온다.

Expected:
- 현재 보고 있는 문항이 정상이어도, 다른 문항에 오류가 있으면 `저장하기` 버튼이 비활성화되어야 한다.

## 15) 메인/미리보기 드롭다운 필터 일관성
1. 문항 3개 이상을 만들고 각 문항에 서로 다른 ID/제목을 입력한다.
2. 메인 드롭다운과 미리보기 드롭다운에서 동일한 검색어(예: 특정 ID 일부 또는 지문 일부)를 각각 입력한다.
3. 검색 결과 목록과 항목 선택 동작을 비교한다.

Expected:
- 두 드롭다운 모두 같은 매칭 규칙(ID+제목+지문 포함, 대소문자 무시, 부분 일치)으로 동일한 항목이 표시되어야 한다.
- 검색 후 항목 선택 시 메인은 편집 문항 전환, 미리보기는 해당 문항으로 스크롤 이동이라는 맥락 차이만 존재해야 한다.

## 16) 레거시 자동저장 이미지 마이그레이션
1. 구버전 키(`json-editor-autosave`)와 레거시 이미지 DB(`json-editor-autosave-assets`)에 자동저장 데이터(이미지 포함)를 준비한다.
2. 현재 버전을 열고 자동 복구가 완료될 때까지 대기한다.
3. 페이지를 새로고침한다.

Expected:
- 첫 진입과 새로고침 이후 모두 이미지가 유지되어야 한다.
- 마이그레이션 실패 상황에서는 레거시 키/DB 경로가 유지되어 이미지 유실이 발생하지 않아야 한다.

## 17) 불러오기 비정상 문항 건너뛰기
1. `questions` 배열에 정상 문항과 비정상 값(`null`, 숫자, 문자열, 배열)을 섞어 JSON 파일을 만든다.
2. 불러오기를 실행하고 가져오기 목록을 확인한다.
3. `정상 문항만` 선택 추가 또는 전체 교체 후 편집 화면에서 문항 수를 확인한다.

Expected:
- 비정상 항목은 앱 오류 없이 자동으로 제외되어야 한다.
- 정상 문항만 가져오기 목록/편집 상태에 반영되어야 한다.

## 18) 검증 경고 칩 점프
1. 문항 3개 이상을 만든다.
2. 현재 문항은 정상으로 두고, 다른 두 문항은 오류 상태로 만든다(예: ID 비움).
3. 현재 문항의 검증 경고 영역에 표시된 오류 문항 칩(`n번 문항`)을 클릭한다.

Expected:
- 저장 버튼은 비활성화 상태를 유지해야 한다.
- 경고 칩 클릭 시 해당 오류 문항으로 즉시 이동해야 한다.

## 19) 드롭다운 검색 중 문항 수정 반영
1. 문항 2개 이상을 만든 뒤, 메인 문항 드롭다운 검색창에 특정 키워드를 입력한다(현재 어떤 문항도 매칭되지 않는 키워드).
2. 검색창은 그대로 둔 상태에서 한 문항 지문을 키워드가 포함되도록 수정한다.
3. 드롭다운 목록 표시가 즉시 갱신되는지 확인한다.

Expected:
- 검색어를 다시 입력하지 않아도, 수정된 문항이 즉시 검색 결과에 나타나야 한다.

## 20) 불러오기 검색 + 비동기 렌더 일관성
1. 문항 수가 많은 JSON/ZIP 파일을 불러오기 모달로 연다(수십 개 이상 권장).
2. 목록이 모두 렌더링되기 전에 불러오기 검색창에 특정 키워드를 입력한다.
3. 렌더링이 진행되는 동안과 완료 후, 목록에 검색 불일치 항목이 섞여 보이는지 확인한다.

Expected:
- 검색 입력 시점 이후에 렌더링되는 항목도 동일한 검색 필터가 즉시 적용되어야 한다.
- 검색 불일치 항목이 뒤늦게 나타나면 안 된다.

## 21) 확인 모달 Enter 키 동작
1. 삭제/초기화 같은 확인 모달을 연다.
2. `취소` 버튼에 포커스를 이동한 뒤 `Enter`를 누른다.
3. 다시 모달을 열고 `확인` 버튼 포커스 상태에서 `Enter`를 누른다.

Expected:
- `취소` 포커스 + `Enter`는 취소로 처리되어야 한다.
- `확인` 포커스 + `Enter`는 확인으로 처리되어야 한다.

## 22) 불러오기 전체 교체 확인
1. 기존 문항이 있는 상태에서 불러오기 모달을 연다.
2. `전체 교체` 버튼을 누른 뒤 확인 모달에서 `취소`를 선택한다.
3. 다시 `전체 교체` 버튼을 누른 뒤 확인 모달에서 `전체 교체`를 선택한다.

Expected:
- `취소` 시 기존 문항이 유지되어야 한다.
- `전체 교체` 확인 시에만 문항이 덮어써져야 한다.

## 23) ZIP 다중 폴더 동일 파일명 점프
1. ZIP 안에 `A/questions.json`, `B/questions.json`처럼 파일명은 같고 폴더만 다른 JSON 2개를 준비한다.
2. 불러오기 모달에서 파일 네비 드롭다운을 열고 각 파일 항목을 번갈아 클릭한다.
3. 목록 스크롤 위치가 올바른 파일 섹션으로 이동하는지 확인한다.

Expected:
- 같은 파일명이라도 폴더가 다르면 서로 다른 섹션으로 정확히 점프해야 한다.

## 24) ZIP 초과 이미지 자동 제외
1. 5MB를 초과하는 이미지가 포함된 ZIP 불러오기 파일을 준비한다.
2. 불러오기 모달을 열고 `전체 교체` 또는 `선택 추가`를 실행한다.
3. 불러오기 진행 문구와 문항 이미지 반영 상태를 확인한다.

Expected:
- 5MB 초과 이미지는 불러오기에서 제외되어야 한다.
- 제외된 이미지 개수가 안내 모달로 표시되어야 한다.

## 25) ZIP 지연 이미지 처리 진행 표시
1. 이미지가 포함된 문항이 여러 개인 ZIP 불러오기 파일을 준비한다.
2. 불러오기 모달을 연 뒤 `전체 교체` 또는 `선택 추가`를 실행한다.
3. 처리 중 상단 상태 문구를 확인한다.

Expected:
- 모달이 먼저 열린 뒤, 실제 가져오기 시점에만 `문항 준비 중... n/m` 진행 문구가 표시되어야 한다.
- 처리 중에는 검색/파일 이동/가져오기 버튼이 비활성화되어야 한다.

## 26) 불러오기 검색 변경 후 숨김 선택 유지
1. 문항이 여러 개인 불러오기 모달을 연다.
2. 검색어 A로 1개 이상 체크한 뒤, 검색어를 바꿔 다른 문항도 체크한다.
3. 마지막 검색 결과에는 처음 체크한 문항이 보이지 않도록 만든 뒤 `선택 추가`를 실행한다.

Expected:
- 현재 숨겨진 체크 항목도 함께 추가되어야 한다.
- 요약 문구에 전체 선택 수가 유지되고, 필요하면 `숨김 선택 n개`가 표시되어야 한다.

## 27) 드롭다운 키보드 선택
1. 메인 문항 드롭다운을 열고 `Tab`으로 목록 항목에 포커스를 이동한다.
2. 포커스된 항목에서 `Enter` 또는 `Space`를 눌러 문항을 선택한다.
3. 전체 미리보기 드롭다운과 불러오기 파일 네비 드롭다운에서도 같은 방식으로 항목 선택을 반복한다.

Expected:
- 세 드롭다운 모두 목록 항목이 키보드 포커스를 받아야 한다.
- `Enter`와 `Space`로 마우스 클릭과 동일하게 선택/이동이 동작해야 한다.

## 28) 모달 내부 드롭다운 Escape
1. 전체 미리보기를 열고 상단 문항 드롭다운을 연다.
2. 검색창에서 `Escape`를 누른다.
3. 불러오기 모달을 열고 파일 네비 드롭다운을 연 뒤 `Escape`를 누른다.

Expected:
- `Escape`는 먼저 드롭다운만 닫고, 미리보기/불러오기 모달 자체는 열린 상태를 유지해야 한다.
- 드롭다운이 닫힌 뒤 포커스는 해당 드롭다운 버튼으로 돌아가야 한다.

## 29) 툴바 호환성 + 접근성
1. `지문` 입력창에 포커스한 뒤 `정답식` 입력창으로 이동하고, 비활성 버튼 구성이 즉시 바뀌는지 확인한다.
2. 정답식 모드에서 비활성 버튼 위에 마우스를 올려 차단 사유 툴팁이 의미 있게 보이는지 확인한다.
3. 정답식 모드에서 `toolbar-action-hint`가 표시되고, 지문 모드에서 숨겨지는지 확인한다.
4. 툴바 탭에서 `ArrowLeft/ArrowRight`, `Home/End`, `Enter/Space` 키 이동이 정상 동작하는지 확인한다.
5. 툴바에서 `\\overline{}`와 `\\underline{}`를 넣고 아래를 확인한다.
   - 지문 미리보기에서 수식 문맥 렌더링이 정상인지
   - 정답식 모드에서 파서 기준 비지원 버튼이 차단되는지
6. 연산 탭의 `합/곱 기호` 그룹에 `\\sum`, `\\prod`가 포함되는지 확인한다.
